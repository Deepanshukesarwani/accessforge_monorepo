### Key design decisions

- `apps/` holds deployable applications. Each app is an independent deployable unit with its own `package.json`, `Dockerfile`, and environment variables.
- `package/` holds internal libraries that are never deployed standalone. They are consumed by apps via pnpm workspace protocol (`"@repo/shared": "workspace:*"`).
- Turborepo ensures that shared packages are built **before** the apps that depend on them (`"dependsOn": ["^build"]`).

## Environment Variables

Copy `.env.example` and fill in values. Each app reads its own `.env` file вЂ” they are never shared across apps.

```bash
cp .env.example apps/backend/.env
# create apps/frontend/.env.local for Next.js
```

### `apps/backend/.env`

| Variable      | Default                                  | Description                          |
|---------------|------------------------------------------|--------------------------------------|
| `PORT`        | `5000`                                   | Port the Express server listens on   |
| `MONGODB_URI` | `mongodb://localhost:27017/secureapi`    | MongoDB connection string            |
| `NODE_ENV`    | `development`                            | `development` / `production`         |
| `JWT_SECRET`  | *(must set)*                             | Secret for signing JWT tokens        |
| `CORS_ORIGIN` | `http://localhost:3000`                  | Allowed CORS origin (frontend URL)   |

### `apps/frontend/.env.local`

| Variable                 | Default                   | Description                         |
|--------------------------|---------------------------|-------------------------------------|
| `NEXT_PUBLIC_API_BASE_URL` | `http://localhost:5000`  | Backend API URL (exposed to browser)|

## Local Development

### 1. Install all dependencies (from repo root)

```bash
pnpm install
```

This installs dependencies for all workspaces in one command. pnpm hoists shared packages and symlinks `@repo/shared` into each app automatically.

### 2. Run everything in parallel (recommended)

```bash
pnpm dev
```

Turborepo starts both `apps/backend` and `apps/frontend` concurrently. Output is prefixed per-app.

| App      | URL                          |
|----------|------------------------------|
| Backend  | http://localhost:5000        |
| Frontend | http://localhost:3000        |
| Health   | http://localhost:5000/api/health |

### 3. Run a single app

```bash
# Backend only
pnpm --filter backend dev

# Frontend only
pnpm --filter frontend dev
```

## Building for Production

```bash
# Build all apps (Turbo resolves dependency order automatically)
pnpm build
```

Turbo builds `@repo/shared` first, then compiles `backend` (outputs to `apps/backend/dist/`) and `frontend` (outputs to `apps/frontend/.next/`).

```bash
# Start backend in production mode
pnpm --filter backend start   # runs: node dist/index.js

# Start frontend in production mode
pnpm --filter frontend start  # runs: next start
```

## Docker вЂ” Deploying Backend & Frontend Separately

The backend and frontend are deployed as independent Docker images. This allows the DevOps team to scale, update, and roll back each service independently.

### Backend Dockerfile

Create `apps/backend/Dockerfile`:

```dockerfile
# в”Ђв”Ђ Stage 1: Build в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
FROM node:20-alpine AS builder

WORKDIR /app

# Install pnpm
RUN npm install -g pnpm@10.13.1

# Copy workspace manifests first (layer cache)
COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY apps/backend/package.json ./apps/backend/
COPY package/shared/package.json ./package/shared/

# Install only production + dev deps needed for the build
RUN pnpm install --frozen-lockfile

# Copy source
COPY tsconfig.json ./
COPY apps/backend/ ./apps/backend/
COPY package/shared/ ./package/shared/

# Build shared package then backend
RUN pnpm --filter @repo/shared build
RUN pnpm --filter backend build

# в”Ђв”Ђ Stage 2: Production image в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
FROM node:20-alpine AS runner

WORKDIR /app

RUN npm install -g pnpm@10.13.1

COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY apps/backend/package.json ./apps/backend/

# Install production deps only
RUN pnpm install --frozen-lockfile --prod

# Copy compiled output
COPY --from=builder /app/apps/backend/dist ./apps/backend/dist

EXPOSE 5000

CMD ["node", "apps/backend/dist/index.js"]
```

Build and run:

```bash
# Build image
docker build -t secureapi-backend:latest -f apps/backend/Dockerfile .

# Run container (pass env vars at runtime вЂ” never bake secrets into the image)
docker run -d \
  --name secureapi-backend \
  -p 5000:5000 \
  -e PORT=5000 \
  -e MONGODB_URI="mongodb+srv://<user>:<pass>@cluster0.xxxx.mongodb.net/secureapi" \
  -e JWT_SECRET="your-super-secret" \
  -e NODE_ENV=production \
  -e CORS_ORIGIN="https://your-frontend-domain.com" \
  secureapi-backend:latest
```

### Frontend Dockerfile

Create `apps/frontend/Dockerfile`:

```dockerfile
# в”Ђв”Ђ Stage 1: Build в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
FROM node:20-alpine AS builder

WORKDIR /app

RUN npm install -g pnpm@10.13.1

COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY apps/frontend/package.json ./apps/frontend/

RUN pnpm install --frozen-lockfile

COPY apps/frontend/ ./apps/frontend/

# Pass the API URL at build time (Next.js bakes NEXT_PUBLIC_* into the bundle)
ARG NEXT_PUBLIC_API_BASE_URL=http://localhost:5000
ENV NEXT_PUBLIC_API_BASE_URL=$NEXT_PUBLIC_API_BASE_URL

RUN pnpm --filter frontend build

# в”Ђв”Ђ Stage 2: Production image в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
FROM node:20-alpine AS runner

WORKDIR /app

ENV NODE_ENV=production

RUN npm install -g pnpm@10.13.1

COPY package.json pnpm-workspace.yaml pnpm-lock.yaml ./
COPY apps/frontend/package.json ./apps/frontend/

RUN pnpm install --frozen-lockfile --prod

COPY --from=builder /app/apps/frontend/.next ./apps/frontend/.next
COPY --from=builder /app/apps/frontend/public ./apps/frontend/public

EXPOSE 3000

CMD ["pnpm", "--filter", "frontend", "start"]
```

Build and run:

```bash
# Build image (pass the real backend URL at build time)
docker build \
  --build-arg NEXT_PUBLIC_API_BASE_URL=https://api.your-domain.com \
  -t secureapi-frontend:latest \
  -f apps/frontend/Dockerfile .

# Run container
docker run -d \
  --name secureapi-frontend \
  -p 3000:3000 \
  secureapi-frontend:latest
```

> **Important:** `NEXT_PUBLIC_*` variables are embedded into the JavaScript bundle at **build time**. You must pass the correct backend URL via `--build-arg` when building the image, not at `docker run` time.

### Docker Compose (local orchestration)

Create `docker-compose.yml` at the repo root:

```yaml
version: "3.9"

services:
  mongo:
    image: mongo:7
    container_name: secureapi-mongo
    restart: unless-stopped
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db

  backend:
    build:
      context: .
      dockerfile: apps/backend/Dockerfile
    container_name: secureapi-backend
    restart: unless-stopped
    ports:
      - "5000:5000"
    environment:
      PORT: 5000
      MONGODB_URI: mongodb://mongo:27017/secureapi
      JWT_SECRET: change-this-in-production
      NODE_ENV: production
      CORS_ORIGIN: http://localhost:3000
    depends_on:
      - mongo

  frontend:
    build:
      context: .
      dockerfile: apps/frontend/Dockerfile
      args:
        NEXT_PUBLIC_API_BASE_URL: http://localhost:5000
    container_name: secureapi-frontend
    restart: unless-stopped
    ports:
      - "3000:3000"
    depends_on:
      - backend

volumes:
  mongo_data:
```

```bash
# Start all services
docker compose up --build

# Stop all services
docker compose down

# Stop and remove volumes (wipes DB data)
docker compose down -v
```

## Shared Packages

### `@repo/shared`

Located at `package/shared/`. Contains TypeScript types and interfaces shared between backend and frontend.

```typescript
// package/shared/src/userTypes.ts
export interface User {
  id: string;
  name: string;
  email: string;
}
```

**How to consume it in any app:**

```json
// apps/backend/package.json or apps/frontend/package.json
{
  "dependencies": {
    "@repo/shared": "workspace:*"
  }
}
```

```typescript
import type { User } from "@repo/shared";
```

pnpm resolves `workspace:*` to the local `package/shared` directory вЂ” no publishing to npm needed.

## Turbo Pipeline

Turborepo controls task execution order. Defined in `turbo.json`:

| Task         | Depends on      | Cached | Description                                  |
|--------------|-----------------|--------|----------------------------------------------|
| `build`      | `^build`        | Yes    | Build all packages; shared packages built first |
| `dev`        | вЂ”               | No     | Run all dev servers concurrently             |
| `lint`       | `^build`        | Yes    | Type-check + lint after build                |
| `type-check` | `^build`        | Yes    | `tsc --noEmit` across all packages           |
| `test`       | `^build`        | Yes    | Run tests, outputs coverage reports          |

`^build` means "run the `build` task in all upstream dependencies first." So if `backend` depends on `@repo/shared`, Turbo builds `@repo/shared` before `backend` automatically.
